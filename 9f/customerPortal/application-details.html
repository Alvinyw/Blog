1. 产品概述

玖借是玖富数科集团推出的智能信用账户产品，为企业提供金融产品模型。企业可快速对接玖借应用产品，建立属于企业的金融业务产品。用户由企业自研客户端进入玖借，通过实名认证、风险评估获得一定的信用额度，即可享受额度分期等服务，链接玖富所有的用户和场景。

2. 适用场景

适用于欲快速对接额度消费借贷金融业务的企业方，包括且不限制于金融机构、旅游类产品、电商平台等。

3. 接入模式

可根据企业需求，可选择一键对接玖借H5全流程模式、部分服务API模式对接辅助运营；也可以通过API全流程对接，更好地留存客户信息。

4. 接口文档-H5

4.1 联合登陆模式

H5 联合登陆模式，商户可通过客户三要素信息调用联合登陆接口，系统后台根据客户在玖借产品流程中所处状态，返回客户所处状态应执行的下一步业务操作页面URL，商户可直接展示该URL，从而跳过前期已操作的业务步骤，简化用户操作，提高用户体验。

4.1.1 接口类型

http请求方式：POST
请求地址：https://open.9fgroup.com/channel/auth

4.1.2 请求参数说明

HTTP请求报文体包含的请求参数如下：

参数名称 中文名称 参数类型 参数长度 是否必填 参数说明
mobile 手机号 String 5 是 客户注册手机号码
name 姓名 String 20 是 客户姓名
cardNo 身份证号 String 128 是 客户身份证号码

示例

{
    "mobile":"18976543877",
    "name":"张三",
    "cardNo":"443777198809129838"

}

4.1.3 响应参数说明

HTTP响应参数中业务参数(data)包含的内容如下：

参数名称 中文名称 参数类型 参数长度 是否必填 参数说明
userID 用户ID String 16 是 玖富开放平台生成的用户ID
url 页面链接 String 128 是 URL跳转地址

示例

{
    "herder":{
        "code":"1028",
        "time":"20190323 18:00:00",
        "sign":"enshjdhjsfggsiowjhjhgehj",
        "channelID":"9823988"，
        "requestNo":"999038377729999340"，
        "version":"1.0.0"
    }

    "body":{
        "userID":"0900239208112376",
        "cellphone":"http://open.9fgroup.com/jiujie/login"
    }
}

4.1.3 状态响应码

响应状态码 说明
000000 执行成功
001000 执行失败
002000 执行异常
303000 黑名单用户
303001 用户不存在
303002 商户编码不存在
303003 商户处于异常状态
303006 身份证一致，手机号不一致
303007 手机号一致，身份证不一致
303008 实名认证失败
400000 鉴权失败

4.2 非联合登陆模式

H5 非联合登陆模式，为商户纯导流模式。商户申请开通该产品对接并审核通过后，系统自动生成与商户唯一对应的URL链接，打开该链接则为玖借产品标准化注册/登陆页。

点击查询对接URL
