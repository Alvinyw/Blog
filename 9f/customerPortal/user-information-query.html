用户信息查询

  • 概述
  • 接口类型
  • 请求参数说明
  • 响应参数说明
  • 响应状态码

1. 概述

本接口用于企业查询所属的客户信息。

2. 接口类型

http请求方式：POST
请求地址：https://open.9fgroup.com/urm/user/query

3. 请求参数说明

HTTP请求报文体包含的请求参数如下：

参数名称 中文名称 参数类型 参数长度 是否必填 参数说明
userID 用户ID String 16 是 玖富开放平台生成的用户ID

HTTP请求报文体包含的请求参数如下：

{ 
    "userID": "0900239208112376" 
}

4. 响应参数说明

HTTP响应参数中业务参数(data)包含的内容如下：

参数名称 中文名称 参数类型 参数长度 是否必填 参数说明
userID 用户ID String 16 是 玖富开放平台生成的用户ID
cellPhone 注册手机号 String 11 是 用户注册时使用的手机号
userName 用户姓名 String 64 否 用户如果未实名则无用户姓名信息
idType 身份类型 String 2 否 用户如果未实名则无用户证件信息
idNo 证件号 String 64 否 用户如果未实名则无用户证件信息
customerID 客户编号 String 64 否 玖富统一客户中心客户编号，存管时需要
extUserID 外部用户编号 String 64 否 外部用户编号，静默注册上传

示例

{ 
    "herder":{ 
        "code":"1028", 
        "time":"20190323 18:00:00", 
        "sign":"enshjdhjsfggsiowjhjhgehj", 
        "channelID":"9823988"， 
        "requestNo":"999038377729999340"， 
        "version":"1.0.0" 
    } 
    "body": {
        "userID":"0900239208112376", 
        "cellphone":"18923348772", 
        "userName":"张三", 
        "idType":"01", 
        "idNo":"", 
        "customerID":"01220112", 
        "extUserID":"12003022" 
    } 
}

5. 响应状态码

除公共的响应状态码外，该接口可能返回的响应状态码说明如下：

响应状态码 说明
010001 用户不存在
