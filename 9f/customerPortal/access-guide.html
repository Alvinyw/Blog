1. 玖富开放平台开发概述

企业在与商务沟通并完成商户准入后，于玖富开放平台登陆，对接口进行申请开通获得接口权限后，可以通过阅读本指南帮助开发接入。

2. 接口类型

如各接口无特殊说明，则为HTTP 接口，POST请求方式

3. HTTP接口请求参数格式

玖富开放平台对外提供的http接口请求参数均包含2部分：
· http报文头(http header)，用于传入固定参数值；
· http报文体，用于传入每个接口所特需的参数值，所需参数由各接口定义，格式为Json。

4. HTTP请求报文头参数说明

接入接口必须在http报文头(http header)中固定传入以下参数：

参数名称 中文名称 参数类型 参数长度 是否必填 参数说明
code 商户编码 String 5 是 玖富开放平台提供给各商户分配的编码
time 请求时间戳 String 20 是 请求时间戳，即请求时间对应的毫秒数
sign 用户姓名 String 128 是 md5(code+time+secret)，其中secret为玖富开放平台分配给商户的密钥
channelID 商户渠道编码 String 10 是 玖富开放平台给商户渠道分配的编码
requestNo 请求流水号 String 32 是 请求流水号，生成规则如下所诉
version 接口版本号 String 10 是 接口版本号，目前统一为：1.0.0

5. 请求流水号生成规则

除公共的响应状态码外，该接口可能返回的响应状态码说明如下：

统一流水号长度固定为 32 位数字，生成规则如下: 商户编码(5 位)+接口编号(4 位)+请求日期(8 位，yyyyMMdd)+自定 义序号(15 位) 说明: 1. 流水号全局不能重复。 2. 接口编号请查看各接口定义

6. HTTP接口响应参数格式

HTTP接口响应参数全部包含在报文体中，报文头中不再包含任何特殊参数。

报文体参数组成如下：

参数名称 中文名称 参数类型 参数长度 是否必填 参数说明
status 响应状态码 String 6 是 各接口返回的响应状态编码
message 响应信息 String 255 是 响应信息详细描述
data 业务参数 String 128 否 各接口返回的业务参数，Json格式，部分可能不回返回次信息，由各接口确定

7. 公共响应状态编码

公共的响应状态码描述如下:

状态码 说明
1 商户编码为空或商户不存在
2 时间戳为空或不合法
3 签名为空或不合法
000000 成功
000001 商户渠道编码为空或商户渠道不存在
000002 请求流水号为空或存在错误
000003 接口版本号为空或格式错误
999999 系统异常

8. 其他说明

所有参数如无特殊说明，则遵循以下规则: · 编码格式均为:UTF-8； · 所有日期属性格式均为:yyyy-MM-dd。； · 所有时间格式均为:HH-mm-ss； · 所有日期时间格式均为:yyyy-MM-dd HH-mm-ss； · 所有年月格式:yyyy-MM。
